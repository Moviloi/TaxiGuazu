# PR-9C — Planning Identity Audit

**Estado:** Borrador de auditoría de identidad  
**Fecha:** 2026-07-13  
**Driver:** Determinar si "Action" (output de Planning) constituye un concepto ontológicamente distinto de entidades existentes.

---

## 1. Action vs Decisión

| Dimensión | Decision (EE) | Action (Planning) |
|-----------|--------------|-------------------|
| Contenido | "readiness = partial" | "ask for origin" |
| Modalidad | Epistémico ("decide que...") | Volitivo ("decide hacer...") |
| Efecto | Cambia estado cognitivo interno | Cambia estado del mundo |

**¿Son diferentes?** ✅ SÍ. Pero la pregunta no es si Action ≠ Decision, sino si Action justifica una CAPA entre Learning y el mundo.

## 2. Action vs Intention (Goal, eliminado)

PR-8C demostró que Goal = Intention = Action de alto nivel. Si Goal fue eliminado por ser idéntico a Action, y Action es el output de Planning, entonces...

**Paradoja:** PR-8 eliminó Goals argumentando que Goal y Action son el mismo tipo ontológico. Planning produce Actions. Entonces:
- Si Goal = Action, y Goal fue eliminado...
- ¿Por qué Planning (que produce Actions) debería existir como capa?

**Consistencia:** Para ser consistentes con PR-8, Planning debe eliminarse. Lo que PR-8 llamó "Action (Planning)" no puede justificar una capa si lo que PR-8 llamó "Goal (Intention)" no la justificó.

## 3. Action vs Efecto Operacional

El sistema operacional ya produce efectos (envía mensajes, ejecuta políticas). Action de Planning sería una representación COGNITIVA de esos efectos.

Pero esta representación es artificial — el sistema operacional no necesita que Planning le "diga" qué hacer. Ya lo sabe.

## 4. Veredicto

**Action no es un concepto ontológicamente distinto dentro del pipeline cognitivo.** Es una duplicación de la lógica operacional existente.

PR-9C confirma PR-9A: **D — Planning debe eliminarse.**
